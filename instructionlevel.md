# Подробный план: `js/modes/tutorial.js`

---

## Архитектура

Единый файл ~600–800 строк. Без внешних зависимостей кроме уже существующих глобалов (`canvas`, `ctx`, `player`, `keys`, `gameMode`, `platforms`, `enemies`, `bullets`, `lives`, `score`, `combo`, `bonusMode`, `bonusShots`, `playerBulletDir`, `altShootMode`, `selectedChar`, `homePlatform`, `shootPlayerBullet(player)` и т.д.).

**Три публичных функции:**
```
initTutorialLevel()     → вызывается из state.js при startMode='tutorial'
updateTutorialMode(dt)  → вызывается из update.js вместо основного update
drawTutorialMode()      → вызывается из draw.js вместо основного draw
```

---

## Переменные состояния

```
_tPhase          — текущая фаза (0..N)
_tSubPhase       — подфаза внутри фазы
_tOverlay        — текст в большом шаре по центру экрана (или null)
_tOverlayTimer   — сколько секунд показывать без нажатия
_tWaitKey        — true = пауза, ждать любую клавишу для продолжения
_tArrow          — { x, y, dir, label } — нарисовать стрелку-указатель
_tBananaz        — массив { x, y, nextChar, label } — точки смены персонажа
_tBossSpawned    — флаг для boss67
_tSwitchCount    — сколько раз уже сменили персонажа
_tDone           — флаг завершения
_tKillCount      — счётчик убийств текущей волны
_tKillTarget     — сколько нужно убить в текущей волне
_tSavedLives     — lives до платформенного сегмента (защита от потери жизни)
_tCharOrder      — ['kuzy','max','dron'] — порядок смены
_tCurrentCharIdx — 0, 1, или 2
```

---

## Структура карты (фиксированный экран, `gameMode='tutorial'`)

**Фон:** `img/bg-avs2.png` (растянуть на весь canvas).

**Платформы:** создаются через `new Platform(...)`, используют те же текстуры что в игре.

Три зоны по горизонтали:
- **Левая (0..40% ширины):** зона движения и базовой стрельбы
- **Центральная (40..70%):** бонусные пули + altShootMode + смена персонажа
- **Правая (70..100%):** платформенный сегмент + босс

Игрок начинает слева на `homePlatform` (широкая низкая платформа во всю ширину экрана на уровне 80% высоты).

```
homePlatform: x=0, y=canvas.height*0.78, w=canvas.width, h=20
```

Дополнительные платформы правой зоны (3 ступени):
```
p1: x=0.68*W, y=0.65*H, w=0.10*W
p2: x=0.76*W, y=0.52*H, w=0.10*W  
p3: x=0.84*W, y=0.38*H, w=0.10*W
```

**Банановые станции** (три штуки, нарисовать через ctx — эмодзи 🍌 размером 40px):
```
banana[0]: x=0.38*W, y=0.70*H  → 'kuzy' → 'max'
banana[1]: x=0.65*W, y=0.70*H  → 'max'  → 'dron'
banana[2]: x=0.92*W, y=0.70*H  → 'dron' → 'kuzy'
```
Рядом с каждой подпись: "→ [имя]". Коллизия через AABB 40×40.

## Последовательность фаз

Фазы переключаются сами — либо по убийству нужного числа врагов (`_tKillCount >= _tKillTarget`), либо после взаимодействия с бананом, либо по кнопке (если `_tWaitKey=true` — любая клавиша продолжает).

---

### Фаза 0 — HUD-брифинг

**Триггер старта:** `initTutorialLevel()` вызывает фазу 0 сразу.

**Механика:** заморозить игрока (`_tFrozen=true`), рисовать полупрозрачный оверлей поверх игры. Показать три блока с подписями стрелками:

1. Стрелка → на блок жизней (❤ × 7(как и во всех уровнях)):  
   *"Это — жизни. Потеряй все семь — игра окончена."*
2. Стрелка → на счёт:  
   *"Очки растут за каждого убитого врага."*
3. Стрелка → на комбо:  
   *"Комбо множит очки если бить без пауз."*

Внизу крупный текст: **"Нажми любую клавишу чтобы начать"**

**Переход:** любая клавиша → фаза 1.

---

### Фаза 1 — Движение (персонаж: Кузя)

**Персонаж:** `kuzy`. Убедиться что `player` инициализирован как `new Player('kuzy','kuzy')`.

**Сообщение сверху:** *"← → — ходить. ↑ — прыгнуть. Зажми прыжок — взлетишь выше!"*

**Задание:** на расстоянии 25% ширины от старта поставить мигающий маркер (круг через ctx). Как только игрок доходит до `x > 0.22*W` — сообщение меняется на:  
*"Подпрыгни! Зажми ↑ дольше — поднимешься выше."*

**Переход:** игрок прыгает хотя бы один раз (`player.isJumping` переходит в false после прыжка) → фаза 2.

**Подсказка прыжков Кузи (показать после прыжка):** *"Кузя разгоняется плавно — чем дольше держишь ↑, тем выше."*

---

### Фаза 2 — Базовая стрельба (Кузя)

**Три волны врагов:**

**Волна 2a:** 2 сирени справа от игрока (правее на 30% экрана), `playerBulletDir='right'`.
- Сообщение: *"Пробел — стрелять. Враги справа!"*
- Переход: оба убиты.

**Волна 2b:** 2 сирени СЛЕВА от игрока.
- Сообщение: *"Враги слева! Нажми ↑ чтобы сменить направление, потом стреляй."*
- Переход: оба убиты.

**Волна 2c:** 1 сирень на платформе выше (поставить временную платформу в центре на высоте 50% экрана).
- Сообщение: *"Враг сверху! Повернись вверх (W) и стреляй вверх."*
- Переход: убит → фаза 3.

---

### Фаза 3 — Бонусные пули

**Предусловие:** на земле (x≈0.32*W) лежит бутылка пива — объект `bottles[]` с обычным механизмом сбора.

**Сообщение:** *"Подбери пиво — получишь бонусные выстрелы!"*

Враги не спавнятся пока бутылка не подобрана.

После подбора (`bonusMode === true`):
- 2 сирени.
- Сообщение: *"Бонусные пули активны! Они больше и быстрее. Стреляй!"*
- Переход: оба убиты → фаза 4.

---

### Фаза 4 — Альтернативная стрельба (altShootMode)

**Действие при старте:** `altShootMode = true`.

**Сообщение:** *"Режим альт-стрельбы! Стрелки теперь меняют направление и движут одновременно. ↓ = вверх + автовыстрел."*

**Пауза** (`_tWaitKey=true`): нажми любую клавишу.

После нажатия — спавн: 1 враг справа, 1 враг сверху, 1 враг слева одновременно.

После победы над всеми тремя:
- `altShootMode = false`
- Сообщение: *"Режим отключён. Далее — смена персонажа!"*
- Пауза → фаза 5.

---

### Фаза 5 — Первая банановая станция (Кузя → Макс)

Отобразить мигающую стрелку над `banana[0]` (x=38%).

**Сообщение:** *"Дотронься до банана чтобы сменить персонажа!"*

**Триггер:** коллизия игрока с `banana[0]` → `player = new Player('max','max')`.

**После смены:**
- Сообщение: *"Это Макс! Его прыжок — одна высокая дуга, удержание не работает. Зато он быстрее стреляет (💩)."*
- Пауза.
- Задание: прыгнуть 2 раза (`_tJumpCount`).
- Сообщение после: *"Видишь? Быстрый и высокий, но высоту не регулируешь."*
- Фаза 6.

---

### Фаза 6 — Прыжок+стрельба (Макс)

**Две сирени** стоят на временной платформе на высоте 45% экрана.

**Сообщение:** *"Эти враги на платформе выше! Запрыгни и стреляй в прыжке."*

**Условие:** убить обоих (находясь в воздухе или нет — не важно, просто убить).

После их смерти: убрать временную платформу. Фаза 7.

---

### Фаза 7 — Платформенный сегмент (Макс)

**Переключить** `gameMode = 'platforms'`. Сохранить `_tSavedLives = lives`.

**Сообщение:** *"Теперь — прыжки по платформам! Упадёшь — вернёшься на старт без потери жизни."*

Три ступени (p1/p2/p3 из раздела карты). На вершине p3 — мигающая звезда (нарисовать через ctx, ★).

**Переход:** игрок достигает p3 (`player.y < 0.42*H`) → победный звук, `lives = _tSavedLives` (восстановить если упал), `gameMode = 'tutorial'`, фаза 8.

**Защита от потери жизни:** в `updateTutorialMode` в фазе 7: если `lives < _tSavedLives` → `lives = _tSavedLives` (перезаписывать каждый кадр).

---

### Фаза 8 — Вторая банановая станция (Макс → Дрон)

Мигающая стрелка над `banana[1]` (x=65%).

**Сообщение:** *"Ещё одна смена! Дрон управляет прыжком иначе."*

**Коллизия с banana[1]** → `player = new Player('dron','dron')`.

**После смены:**
- Сообщение: *"Это Дрон! Его пули — вихрь (🌀). Зажми прыжок — он летит выше. Сравни с Максом!"*
- Пауза.
- Задание: снова пройти те же три платформы (оставить p1/p2/p3).
- После достижения p3 → фаза 9.

Не нужно защищать жизни здесь — `gameMode='platforms'` уже активен, просто не сбрасывать до конца сегмента.

---

### Фаза 9 — Босс (Дрон)

`gameMode = 'tutorial'` (выключить платформенную физику).

**Спавн boss67:** использовать `enemy67` по аналогии с другими уровнями. Установить `enemy.hp = 10`, `enemy.maxHp = 10`.

Сообщения не показывать — просто появляется босс (как в настоящей игре). Показать полосу HP под боссом через `drawStyledBossHpBar(...)`.

**Переход:** босс убит (`enemies.length === 0 || enemy67 === null`) → небольшая пауза 1 сек → сообщение *"Готово!"* → фаза 10.

---

### Фаза 10 — Финал

Мигающая стрелка над `banana[2]` (x=92%).

**Коллизия с banana[2]** → вернуть Кузю: `player = new Player('kuzy','kuzy')`.

**Сообщение:**  
*"Обучение пройдено! Ты знаешь всё необходимое. Теперь тебе открыта кампания."*

Пауза → любая клавиша → `localStorage.setItem('bh_tutorial_done_v1','1')` → `showLevelComplete()`.

## Система смены персонажа

В `updateTutorialMode(dt)` каждый кадр:

```javascript
_tBananaz.forEach(b => {
    if (playerIntersectsZone(b.x, b.y, 40, 40) && currentPhaseIsAppropriateBanana(b)) {
        switchToChar(b.nextChar);
        advancePhase();
    }
});
```

`switchToChar(type)`:
1. `selectedChar = type`
2. Сохранить `x, y` текущего игрока
3. `player = new Player(type, type)`
4. Восстановить позицию

---

## Визуальные хелперы (только ctx, никаких новых картинок)

**`_tDrawArrow(x, y, dir, label)`** — рисует треугольную стрелку + текст. `dir` = `'up'|'down'|'left'|'right'`. Анимирует offset через `Math.sin(time)`.

**`_tDrawOverlay(text)`** — полупрозрачный rect (`rgba(0,0,0,0.6)`), белый текст, внизу: *"[Любая клавиша — продолжить]"* если `_tWaitKey=true`.

**`_tDrawBanana(b)`** — цикл по `_tBananaz`: рисовать `ctx.fillText('🍌', b.x, b.y)`, если активная (не использована) — добавить мигающий ореол. Неактивные — серые (`ctx.globalAlpha = 0.3`).

**`_tDrawHudHighlight(region, text)`** — обводка прямоугольником и выноска. Используется только в фазе 0.

## Блокировка кампании и кнопка туториала

### В `index.html`

1. Кнопка туториала внизу по центру, отдельно от сетки уровней:
   ```html
   <div id="tutorial-btn-wrap" style="position:absolute; bottom:16px; left:50%; transform:translateX(-50%)">
     <button id="btnTutorial">📖 Обучение</button>
   </div>
   ```
   Визуально: другой цвет фона (например, `#4a7c4e`), рамка. Не входит в нумерацию уровней.

2. Обработчик клика по кнопкам уровней кампании (до запуска уровня):
   ```javascript
   if (!localStorage.getItem('bh_tutorial_done_v1')) {
       showTutorialHint(); // показать всплывашку
       return; // не запускать уровень
   }
   ```

3. Всплывашка `showTutorialHint()` — простой div поверх, текст: *"Сначала пройди обучение! Нажми кнопку ниже."*, кнопка «Закрыть».

4. После прохождения туториала (`localStorage` ключ установлен): `showTutorialHint` не показывается, все кнопки уровней кликабельны.

## Что НЕ делать

- Никаких NPC-персонажей стоящих на земле
- Никаких новых изображений (только эмодзи через `ctx.fillText`)
- Никаких таймеров вместо триггеров по действию игрока (фазы переключаются только когда игрок выполнил задание)
- Не трогать файлы `state.js`, `update.js`, `draw.js`, `progression.js`, `index.html` — хуки там уже есть

## Порядок разработки

1. `initTutorialLevel()` — инициализация всех переменных, создание платформ и banana-точек, spawn игрока
2. Фаза 0 (HUD)
3. Фазы 1–2 (движение + стрельба, самые базовые)
4. Фаза 3 (бонусные пули)
5. Фаза 4 (altShootMode)
6. Механика смены персонажа (banana коллизия) → тест
7. Фазы 5–6–7 (Макс, платформы)
8. Фазы 8–9 (Дрон, босс)
9. Фаза 10 + финал + localStorage
10. Блокировка кампании в index.html
